# Codex Commands and Prompt Patterns

These are representative prompt patterns used during development.

## Backend patterns
- "Implement state transition validation in service layer, not routes."
- "Add explicit re-edit endpoint for `REJECTED -> DRAFT` and update tests."
- "Add integration test for `DRAFT -> SUBMITTED -> APPROVED` through API routes."

## Frontend patterns
- "Add confirmation dialog before admin approve/reject actions."
- "Implement item edit mode and receipt attach/update for selected item."
- "Add reusable loading overlay component and use it in report detail page."

## Maintenance patterns
- "Stage and commit current changes with descriptive commit message."
- "Refresh README for 5-minute reviewer setup path."
